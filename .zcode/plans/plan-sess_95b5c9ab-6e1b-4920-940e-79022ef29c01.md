## 修复计划：3 个问题，改 2 个文件

### 问题 1：资金流向全为 0 — 改 `data.py:729`

**根因**: `fund_flow_daily_async()` 调用东财 `push2his` API 时使用 `_get_json()`，但该函数不设 `Referer` 头，`push2his` 子域名要求 `Referer` 才能返回数据。`push2` 子域名不需要。

**修复**: 将 `fund_flow_daily_async` 从 `_get_json` 改为直接 `s.get()` 并传入 `headers={"Referer": "https://quote.eastmoney.com/"}`，与同类函数 `northbound_flow_async`(line 789) 保持一致。

```python
# 改前
d = (await _get_json("https://push2his.eastmoney.com/api/qt/stock/fflow/daykline/get", params={...})).get("data")

# 改后
s = await get_async_session()
async with s.get("https://push2his.eastmoney.com/api/qt/stock/fflow/daykline/get",
    params={...}, headers={"Referer": "https://quote.eastmoney.com/"}) as r:
    d = (await r.json()).get("data")
```

**影响范围**: `batch_hk_capital_flow_async` + `cn_fund_flow_minute_async` 都经此函数，两者同时修复。

### 问题 2：基本面评分失真 — 改 `recommend_hk.py:118-142`

**根因**: `fb_score()` 对以下情况没有扣分：
- PE 为负（亏损企业）→ 无惩罚
- ROE 为负 → 无惩罚（`0 < roe < 3: -0.5` 条件不命中负值）
- 净利同比 `ny` 严重为负 → 完全未使用 `ny` 字段

以 01548 金斯瑞为例：PE=-7.56，净利同比=-118.65%，ROE=-12.87%，但营收增 61.4% 和毛利率 57.65% 把分拉到 5/5。

**修复**: 在 `fb_score` 中追加扣分逻辑：

| 条件 | 变化 |
|------|------|
| 新增 `pe < 0` | `s -= 1.5`（亏损企业重大扣分） |
| 新增 `pe < 5` | `s -= 0.5`（过低PE可能含陷阱） |
| 新增 `roe < 0` | `s -= 1.0`（股东亏损） |
| 新增 `ny < -50` | `s -= 1.0`（净利大幅恶化） |
| 新增 `ny < 0` | `s -= 0.5`（净利小幅下滑） |

### 问题 3：MACD 柱空白 — 改 `data.py` 或 `recommend_hk.py:223-237`

**根因**: `chan_score()` 中 MACD 字典 key 名与 `calc_macd()` 返回值不匹配：

| `calc_macd` 返回的 key | `chan_score` 使用的 key | 结果 |
|---|---|---|
| `macd_hist` | `histogram` (不存在) | fallback 到 `m["macd"] - m["signal"]` |
| `dif` | `macd` (不存在) | KeyError → 被 try/except 静默吞掉 |
| `dea` | `signal` (不存在) | 同上 |

**修复**: 将 `chan_score` 中的 MACD key 名修正为 `calc_macd` 的实际返回字段：

```python
# 改前
hi = m.get("histogram", m["macd"] - m["signal"])
ph = pm.get("histogram", pm["macd"] - pm["signal"])

# 改后
hi = m.get("macd_hist", (m["dif"] - m["dea"]) * 2)
ph = pm.get("macd_hist", (pm["dif"] - pm["dea"]) * 2)
```
